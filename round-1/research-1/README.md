## Full Summary

This research comprehensively maps Prolog's exception signals and goal-failure patterns to five structurally distinct failure modes in text-to-first-order-logic pipelines: (1) Lexical Mismatch (predicate name synonymy), (2) Arity Mismatch (wrong argument count), (3) Missing Domain Fact (valid predicate but absent ground atoms), (4) Ontological Category Violation (entity type incompatibility), and (5) Quantifier Scope Conflict (∀∃ vs. ∃∀ ambiguity). The research synthesizes repair strategies from four seminal systems: ARGOS (Cotnareanu et al. ICLR 2026) handles Type 3 via iterative abductive augmentation using SAT solver backbone feedback (18.4 average LLM calls per problem); NLProlog (Weber et al. ACL 2019) addresses Type 1 via soft unification with learned embeddings (cosine similarity on SentVec); RECOVER (Cornelio & Diab 2024) demonstrates ontology-driven type-specific failure detection with explicit logical rules; and abductive logic programming literature provides principled fact generation. The output is a concrete typed-failure detector design with: (a) exception-to-failure-type decision tree with heuristics (semantic similarity >0.8 for Type 1, ontology type-checking for Type 4, proof-tree analysis for Type 5); (b) five specialized repair prompts tailored to failure semantics (bridge axioms, FOL restructuring, minimal abduction, entity re-typing, scope re-annotation); and (c) a rigorous benchmark evaluation plan using RuleTaker (100K examples per difficulty level D0–D5, systematic composition) and CLUTRR (kinship inference, 3-8 reasoning hops, human performance 70% for k≤3, 40-50% for k>3, noise robustness testing). Key finding: ARGOS uses undifferentiated prompting (generic "fill in blank" for all failures, 81% accuracy on FOLIO, 84% on CLUTRR); NLProlog requires end-to-end training; RECOVER demonstrates type-specific detection principle but requires manual ontology engineering. A unified typed detector promises improved precision by routing each failure type to specialized repair logic rather than generic fallback.
