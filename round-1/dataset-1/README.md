## Full Summary

Two peer-reviewed neuro-symbolic reasoning benchmark datasets are acquired from HuggingFace and standardized to the exp_sel_data_out schema.

**ProofWriter** (tasksource/proofwriter, ACL 2021): 230,242 examples across train/test splits. Each example includes a natural-language theory (facts + rules), a yes/no query, and gold proof traces in the `allProofs` field — the primary reason this dataset was chosen over plain RuleTaker. Fields mapped: input = 'Theory: {theory}\nQuery: {question}', output = 'true'/'false', metadata includes reasoning depth (QDep), num_facts, num_rules, and truncated allProofs (gold proof trace). Reasoning depth ranges from 0 to 5+, covering both shallow lookup and deep multi-hop chains.

**CLUTRR** (kendrivp/CLUTRR_v1_extracted, EMNLP 2019): 95,672 examples across train/test splits. Each example is a short kinship narrative story with a relational query (who is X to Y?) and an explicit proof_state chain encoding the multi-hop kinship inference path. Fields mapped: input = 'Story: {story}\nQuery: {query}', output = target kinship term (e.g. 'grandson'), metadata includes proof_state (gold chain), f_comb (relation composition), task_name, and reasoning depth derived from f_comb hop count.

**Why these two datasets**: ProofWriter supersedes RuleTaker by adding gold proof traces essential for evaluating trace-graph auditing. CLUTRR provides complementary relational/kinship reasoning with a structurally different proof format (explicit entity chains vs. rule firing sequences), together covering both propositional and relational FOL reasoning modes required by the neuro-symbolic pipeline.

**Schema**: exp_sel_data_out format — top-level {datasets: [{dataset, examples: [{input, output, metadata_*}]}]}. Schema validation passed (100% of records). Total: 325,914 examples.

**Files**: full_data_out.json split into 5 parts under 100MB each in full_data_out/ directory; mini_data_out.json (6 examples); preview_data_out.json (6 examples, strings truncated).

**Reproducibility**: data.py loads from temp/datasets/ using robust line-by-line JSON parsing (handles truncated downloads). pyproject.toml pins loguru==0.7.3. Python 3.12 required.
