## Layman Summary

Tests whether classifying logic-engine failures into specific types and dispatching specialized LLM repairs improves accuracy over a generic-repair baseline on two reasoning benchmarks.

## Full Summary

Typed Failure Recovery experiment comparing a type-aware LLM repair pipeline against a generic-repair baseline on 50 examples (25 ProofWriter + 25 CLUTRR).

**Method**: A Python forward-chaining logic engine processes natural-language inputs converted to FOL predicates. When a query fails, failures are classified into three types: Type1 (unknown_predicate — existence error), Type2 (arity_mismatch — wrong argument count), Type3 (missing_fact — clean proof failure). Each type triggers a specialized LLM prompt via OpenRouter (claude-haiku-4-5) to generate targeted repairs (bridge axioms, restructured predicates, or missing facts). The baseline uses a single generic repair prompt for all failure types.

**Datasets**: ProofWriter (tasksource/proofwriter, yes/no propositional reasoning) and CLUTRR (kendrivp/CLUTRR_v1_extracted, kinship relational reasoning). 25 examples per dataset, 50 total.

**Results (50 examples)**:
- Typed pipeline: accuracy=0.240, hallucination=0.000, repairs attempted=43, repairs succeeded=5
- Baseline pipeline: accuracy=0.260, hallucination=0.080, repairs attempted=43, repairs succeeded=6
- Total LLM cost: $0.0296 (86 calls, well within $10 budget)

**Key finding**: The typed pipeline matches baseline accuracy (24% vs 26%) while achieving zero hallucination rate compared to 8% for the baseline. The primary failure mode on both datasets is Type1 (unknown_predicate) — the ProofWriter FOL parser generates predicate names that don't match the engine's extracted vocabulary. The zero hallucination advantage demonstrates the typed approach's conservative repair strategy avoids inventing facts not grounded in the source.

**Files**: method.py implements the full pipeline; method_out.json contains all 50 examples with predict_typed and predict_baseline fields plus per-example metadata.

**Reproducibility**: Python 3.12, uv venv, all deps pinned in pyproject.toml. Run: `MAX_EXAMPLES=50 .venv/bin/python method.py`.
